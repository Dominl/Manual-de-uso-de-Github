📝 ¿Qué son los Issues?

Los Issues en GitHub permiten reportar errores, proponer mejoras o hacer preguntas. Son muy útiles para organizar el trabajo y dar seguimiento a tareas dentro del repositorio.

También pueden asignarse, etiquetarse y vincularse a Pull Requests.

🔒 Ramas protegidas

Una rama protegida en GitHub evita que se hagan cambios directos sin pasar por revisiones. Se usa para mantener el código estable, especialmente en main o master.

Permite aplicar reglas como:

  • Evitar commits directos
  • Requerir pull requests
  • Obligar revisiones de código
  • Ejecutar tests automáticos antes del merge

📂 GitHub Projects

Los Projects son tableros tipo Kanban o tablas donde puedes organizar tus tareas, issues y pull requests.

Son útiles para planificar y visualizar el progreso de tu equipo o tus propios avances.

🌐 GitHub Pages

GitHub Pages permite alojar sitios web estáticos directamente desde tu repositorio. Puedes usar HTML, CSS o generadores como Jekyll.

Para activarlo:

  1. Ir a Settings > Pages
  2. Seleccionar la rama y carpeta (por ejemplo, main y /)
  3. Guardar y visitar la URL generada por GitHub
